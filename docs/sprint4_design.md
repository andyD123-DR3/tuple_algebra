## 1. Problem Statement

Sprint 3 delivered `partition(N)`. Sprint 4 connects it to per-group
optimisation:

**Given a partition value, generate per-group subspaces, solve them
independently, and assemble a composite plan.**

## 3. Architecture

### Layer 1: Partition-rooted hierarchy (`tree_space.h`)

This layer is **partition-rooted, not generic**. It assumes:
- The root is a `partition_desc<N>` over a fixed index set of N items.
- Groups are identified by canonical labels 0..K-1 (RGS invariant).
- Lane membership is determined by `group_lanes()`.
- Maximum group count = N.
- Root features are pairwise co-membership (from partition_desc).

A truly generic root type would require a separate grouping-traits
abstraction. That generalisation is explicitly deferred.

Contains:
- `tree_point<N, ChildPoint>` -- composite point
- `tree_space<N, ChildFactory, Filter>` -- composite space
- `tree_bridge<N, ChildBridgeFactory>` -- composite encoding
- `group_lanes()`, `for_each_lane_in_group()` -- utilities
- `make_flat_tree_space()` -- degenerate adapter

Dependencies: `concepts.h`, `partition.h`, `feature_bridge.h`.

### Layer 2: Reduction specialisation (`reduction_tree_space.h`)

Contains:
- `make_group_properties()` -- subset + reindex properties
- `make_fusibility_filter<N>()` -- legality from fingerprints
- `make_reduction_child_factory()` -- builds per-group opt space
- `make_reduction_tree_space()` -- convenience assembler

Dependencies: `tree_space.h`, `reduction_space.h`, `reduction_properties.h`.

### New overload in existing header (`reduction_space.h`)

```cpp
// New: build space from properties directly (no tuple_reduction needed).
auto make_reduction_opt_space(const reduction_properties_t& props);
```

This is the only change to an existing header. 10 lines. Required because
the child factory has group-level properties, not a tuple_reduction.

## 4. Key Types

### 4.1 tree_point<N, ChildPoint>

```cpp
template <std::size_t N, typename ChildPoint>
struct tree_point {
    partition_value<N> root;
    std::vector<ChildPoint> group_plans;  // indexed by canonical label

    std::size_t num_groups() const { return group_plans.size(); }

    // Equality requires ChildPoint to be equality-comparable.
    // This is a deliberate constraint: tree_point is a value type
    // used in sets/maps for deduplication. If ChildPoint is not
    // comparable, use a custom comparator instead.
    bool operator==(const tree_point&) const = default;
};
```

N is the partition size (template parameter, not runtime). ChildPoint
is the child space's point_type (typically std::tuple<...>).

**Why std::vector?** Number of groups varies (1..N) per root value.
Deliberate runtime-first choice. Upgrade path: fixed-capacity
small_vector<ChildPoint, N> if constexpr enumeration is needed later.

### 4.2 tree_space<N, ChildFactory, Filter>

```cpp
template <std::size_t N, typename ChildFactory, typename Filter>
struct tree_space {
    static constexpr std::size_t partition_size = N;
    using root_value_type  = partition_value<N>;
    using child_space_type = /* deduced, see below */;
    using child_point_type = typename child_space_type::point_type;
    using point_type       = tree_point<N, child_point_type>;

    std::string_view name_;   // must point to static-storage or
                              // caller-owned string with sufficient lifetime
    partition_desc<N> root_desc_;
    ChildFactory child_factory_;
    Filter filter_;

    std::string_view space_name() const { return name_; }
    std::size_t cardinality() const;

    template <typename F>
    void enumerate(F&& fn) const;
    // The tree_point passed to fn is a temporary. The reference is
    // valid only for the duration of the callback invocation.
    // Callers that store points must copy (tree_point copies its vector).
};
```

**ChildFactory contract (exact, single signature):**

```cpp
child_space_type factory(
    const partition_value<N>& root,
    std::size_t group_label,
    std::span<const std::size_t> lane_indices) const;
```

Contract requirements:
- **Pure and deterministic.** Given the same arguments, the factory must
  return a child space with the same cardinality, same points, same
  descriptor configuration. Mutable external state is forbidden.
- **Span lifetime.** The `lane_indices` span is valid only for the
  duration of the factory call. The factory must not retain the span,
  any pointers into it, or any iterators over it. Copy the indices
  if needed.
- tree_space calls `group_lanes()` internally and invokes the factory
  per group. The factory does not call `group_lanes` itself.

**Filter contract:**

```cpp
bool filter(const partition_value<N>& root) const;
```

Contract requirements:
- **Pure and deterministic.** Same root always gives same result.
- Called by both cardinality() and enumerate(). If filter is
  non-deterministic, cardinality and enumeration will disagree.

**child_space_type deduction:**

```cpp
using child_space_type = std::invoke_result_t<
    const ChildFactory&,
    const partition_value<N>&,
    std::size_t,
    std::span<const std::size_t>>;
```

The factory must return the same type for every invocation.
This is enforced by the template deduction; a factory returning
different types per group would not compile.

### 4.3 tree_bridge<N, ChildBridgeFactory>

Feature encoding for composite points. **Per-group bridges.**

```cpp
template <std::size_t N, typename ChildBridgeFactory>
struct tree_bridge {
    partition_desc<N> root_desc_;
    ChildBridgeFactory child_bridge_factory_;
    std::size_t child_feature_width_;  // cached, all groups same width

    std::size_t num_features() const {
        // root pairwise: N*(N-1)/2
        // child blocks: N * child_feature_width
        return root_desc_.feature_width() + N * child_feature_width_;
    }

    template <typename ChildPoint>
    std::vector<double> encode(const tree_point<N, ChildPoint>& pt) const;
};
```

**Why ChildBridgeFactory, not a single ChildBridge?**

Different groups have different conditional_dim activation states.
A bridge built from a plus-group encodes tree_shape as active (one-hot
with a real slot). A bridge from a non-associative group encodes it
as inactive (all zeros). The feature WIDTH is stable (conditional_dim
guarantee), but the VALUES differ. Using one group's bridge for
another group's point produces **wrong features**.

The ChildBridgeFactory builds a bridge from a child space:

```cpp
child_bridge_type bridge_factory(const child_space_type& space) const;
```

For reduction spaces, this is simply `default_bridge(space)`.

encode() works as follows:
1. Write root pairwise features (from partition_value).
2. For each group k in 0..pt.num_groups()-1:
   a. Reconstruct child space: call child_factory_(pt.root, k, lanes_k).
   b. Build bridge: call child_bridge_factory_(child_space).
   c. Encode group_plans[k] through that bridge into block k.
3. Zero-fill blocks num_groups..N-1.

**Performance note:** This reconstructs child spaces and bridges during
encoding. For Sprint 4 this is acceptable (child construction is cheap).
If encoding becomes a hotspot, memoisation keyed on (root, group_label)
is the optimisation path.

**MaxGroups invariant:** `N` is the maximum number of groups.
For any legal partition_value<N>, num_groups() <= N. This is guaranteed
by the RGS representation (labels are in 0..N-1). An assert in encode()
verifies pt.num_groups() <= N in debug builds.

### 4.4 group_lanes

No heap allocation. Caller provides stack buffer.

```cpp
// Span version: writes indices into caller buffer, returns count.
template <std::size_t N>
std::size_t group_lanes(
    const partition_value<N>& pv,
    std::size_t group_label,
    std::span<std::size_t> out_indices);

// Callback version: zero allocation.
template <std::size_t N, typename F>
void for_each_lane_in_group(
    const partition_value<N>& pv,
    std::size_t group_label,
    F&& fn);  // called with each lane index in order
```

Implementation: single pass over pv.labels, O(N).

### 4.5 make_flat_tree_space

```cpp
template <typename ChildSpace>
auto make_flat_tree_space(std::string_view name, ChildSpace child);
```

Returns a tree_space<1, ...> where:
- Root: partition_desc<1>, cardinality 1, one value [0].
- Factory: ignores all arguments (root, group_label, lane_indices),
  always returns the stored child space. The lane_indices={0} is a
  formal satisfaction of the contract with no semantic content.
- Filter: always returns true.
- Cardinality = child.cardinality().

**Contract:** Behavioural equivalence under projection, NOT
representational identity. Point types differ (tree_point<1,...>
vs raw child point). Feature schemas differ (partition_desc<1>
has 0 pairwise features, but the tree_bridge wrapper adds structural
overhead). A flat adapter for a 5-lane problem uses partition_desc<1>,
NOT partition_desc<5> -- the two are API-compatible, not
feature-compatible.

## 5. Reduction helpers (Layer 2)

### 5.1 make_group_properties

```cpp
reduction_properties_t make_group_properties(
    const reduction_properties_t& full_props,
    std::span<const std::size_t> lane_indices);
```

**Full reindexing specification.** This function must preserve and
reindex every per-lane property, not just aggregate booleans:

- `lane_count` = lane_indices.size()
- For each i in 0..lane_count-1, copy from full_props at
  lane_indices[i]:
  - `lane_associative[i]`
  - `lane_commutative[i]`
  - `lane_has_identity[i]`
  - `lane_idempotent[i]`
  - `lane_identity_transform[i]`
  - `lane_exact_associative[i]`
  - `lane_exact_commutative[i]`
  - `lane_transform_cost[i]`
  - `lane_fingerprint[i]`
- Recompute all aggregate booleans from the reindexed per-lane arrays:
  - `all_associative = all of lane_associative[0..lane_count)`
  - `all_commutative = all of lane_commutative[0..lane_count)`
  - etc.
- `all_fusible()` and `fusible(i,j)` must work correctly on the
  reindexed fingerprints.

### 5.2 make_fusibility_filter

```cpp
template <std::size_t N>
auto make_fusibility_filter(const reduction_properties_t& props);
```

Returns callable `(partition_value<N>) -> bool`:
- True iff every co-grouped pair satisfies `props.fusible(i,j)`.
- All-fusible: ALL Bell(N) partitions pass.
- All-different: only singletons pass.

**Captures props by value** (or const ref to caller-owned object with
sufficient lifetime). Pure and deterministic.

### 5.3 make_reduction_child_factory

```cpp
template <typename... Lanes>
auto make_reduction_child_factory(
    const ct_dp::algebra::tuple_reduction<Lanes...>& red);
```

Returns a callable matching the factory contract:
- Receives (root, group_label, lane_indices).
- Calls make_group_properties(full_props, lane_indices).
- Calls make_reduction_opt_space(group_props) (the NEW overload).
- Returns the .space member.

### 5.4 make_reduction_tree_space

```cpp

auto make_reduction_tree_space(
    const ct_dp::algebra::tuple_reduction<Lanes...>& red);
```

## 6. Design Decisions

### D4.1: Child spaces all have the same type

Guaranteed by conditional_dim being runtime-parameterised. Different
groups have different active/inactive flags but the same C++ type.
Child cardinality varies (active group: 84, inactive group: 7).

### D4.2: Fixed-width feature encoding with per-group bridges

N child blocks. Block order = canonical partition label order.
Each block encoded through a bridge built from that group's actual
child space. Feature WIDTH is stable (conditional_dim guarantee).
Feature VALUES vary per group (correctly reflecting activation state).

### D4.3: tree_space does not own a solver

Nested_solve is a pattern, not a type.

### D4.4: Flat is behavioural equivalence, not identity

### D4.5: Legality is pure filtering, not canonical collapse

All-fusible means all partitions legal. Dominance pruning is future.

### D4.6: Nested solve exact only for separable objectives

### D4.7: Partition-rooted, not generic

tree_space.h assumes a partition root. The template parameter N is the
partition size. Root features are pairwise co-membership. Root
enumeration uses partition_desc<N>. There is no separate RootBridge --
encoding uses partition_desc::write_features() directly. A generic
root would require a grouping-traits abstraction; that is deferred.

### D4.8: point_at deferred

### D4.9: tree_point equality constrains ChildPoint

tree_point::operator== is defaulted, requiring ChildPoint to be
equality-comparable. This is intentional: tree_points are value types
used in deduplication tests and set-based comparisons. If a future
ChildPoint is not comparable, provide a custom comparator.

### D4.10: name_ is std::string_view with static-storage requirement

tree_space::name_ is string_view. Callers must pass string literals
or pointers to static/program-lifetime storage. This matches the
existing convention in descriptor_space and partition_desc. Do not
pass temporaries.

## 9. Enumeration implementation notes

```
for each root_value in root_desc (i = 0..Bell(N)-1):
    if not filter(root_value): continue
    K = root_value.num_groups()

    enumerate_groups(root_value, group=0, K, partial=[], callback):
        if group == K:
            callback(tree_point{root_value, copy(partial)})
            return

        // Build lane indices on the stack
        std::array<std::size_t, N> lane_buf;
        auto count = group_lanes(root_value, group, lane_buf);
        auto lanes = std::span{lane_buf.data(), count};

        child_space = factory(root_value, group, lanes)
        child_space.enumerate([&](const auto& child_pt) {
            partial.push_back(child_pt)
            enumerate_groups(root_value, group+1, K, partial, callback)
            partial.pop_back()
        })
```

The tree_point passed to the outer callback is constructed by copying
`partial`. The callback receives a value, not a reference to reused
storage. Callers may freely store the tree_point.

Stack depth: O(N). Factory calls per root: K (one per group).
Total factory calls: sum over legal roots of num_groups(root).

## 10. Risk areas

1. **Uniform child type.** If different groups need different space
   types, tree_space won't compile. Mitigated by conditional_dim
   handling property variation at runtime within a fixed type.

2. **Feature size.** N=12: 66 pairwise + 12*5 child = 126 features.
   Manageable for SVR/MLP.

3. **Large cardinality.** Mitigated by legality filtering and nested
   solve. For 5-lane stats: 10 legal partitions, manageable.

4. **Repeated factory/bridge construction.** cardinality(), enumerate(),
   and encode() all reconstruct child spaces. Acceptable for Sprint 4
   (construction is cheap). Memoisation is the upgrade path.

5. **tree_point heap allocation.** std::vector per point. Acceptable
   at runtime search time. Upgrade: small_vector<N>.

6. **Cardinality overflow.** Unchecked std::size_t. Not a concern at
   N<=12 but should be checked if scope grows.

7. **span lifetime in factory.** Documented: span is ephemeral.
   Factory must not retain it. Violation causes silent UB.

8. **string_view lifetime in name_.** Documented: must point to
   static or program-lifetime storage. Matches existing convention.
